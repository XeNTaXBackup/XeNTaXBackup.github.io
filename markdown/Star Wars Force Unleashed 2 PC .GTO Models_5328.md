You have to change your Windows regional settings, if your problem persists with all converted models which you try to import into max.
Go to windows -> Control Panel -> Regions and settings -> Additional Settings (bottom button).
In the Regional Settings of Windows, the Decimal separator must be set to "." instead of "," so that, when one use the GTO Ripper, data in OBJ files are written with "." (ie "1.20" instead of "1,20") so the import plugin should correctly recognize the vertex coordinates of the mesh .

Try to use GTO ripper, it should work fine now.
